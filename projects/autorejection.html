Load Cell PCB
These are the renderings of the auto-rejection machine and load cell controller PCB. This machine can weigh 50+ cans of beer as they come off a canning line. If the cans are under- or overfilled, they will be ejected from the canning line by the small puck on the left side of the machine. The star wheel design allows for precise control over the ejection process, ensuring the cans have enough time to fully stop for each measurement. Achieving that stability was crucial to the machine's performance and reliability, and it was one of the most challenging aspects of the project.
The load cell controller PCB is built around an HX711 load cell amplifier with an ATmega328P microcontroller to take consistent weight samples. The PCB is designed to be compact and easily integrated into the machine's enclosure, with mounting holes that align with the Raspberry Pi's mounting holes. The controller reads the weight measurements from the load cell and communicates with the Node-RED automation system to determine whether each can should be accepted or rejected. Node-Red also hosts a web server accessible via Wi-Fi from the Raspberry Pi that provides real-time weight data, calibration settings, and historical records.

Overview

A Raspberry Pi-based system for industrial process automation, UI integration, and data management. Integrates with physical hardware via Arduino microcontrollers and exposes a live dashboard through Node-RED's UIBuilder module.

Flow logic is defined visually in Node-RED and backed by JavaScript processing scripts. Supports standalone operation and networked deployment modes.

Learnings

Components

  • Node-RED flows for visual automation logic
  • Arduino hardware modules (standalone and networked variants)
  • JavaScript file processing scripts
  • UIBuilder dashboard for real-time monitoring
  • AWS IoT integration for data storage and over the air updates.

Hardware

  • Arduino microcontrollers for physical I/O
  • StandAlone and WildGoose designs for the weight measurement and rejection systems.
  • Raspberry Pi for networked deployment and control through two separate wifi networks.
