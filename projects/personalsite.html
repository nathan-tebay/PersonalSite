PersonalSite

This site — a personal portfolio and documentation platform built from scratch with vanilla HTML, CSS, and JavaScript

View on GitHub

Overview

A hand-rolled personal site with no frameworks or build tools. Layout, theming, navigation, blog, and links are all implemented in plain HTML, CSS, and vanilla JavaScript served by a containerised BusyBox httpd instance.

Blog posts and links are stored in an S3-compatible object store (AWS S3 in production, MinIO locally) and managed through a password-protected admin panel backed by POSIX shell CGI scripts. The nav panel is generated dynamically by a shared layout.js script injected on every page, keeping navigation consistent without a build step.

Stack

HTML5 CSS3 Vanilla JS Shell / CGI BusyBox httpd S3 / MinIO Podman

Key Features

  • Dynamic nav and header injected via a shared layout script — no build step
  • Multi-theme colour picker with WCAG-derived text contrast, persisted to localStorage
  • Blog system: draft/publish workflow, WIP placeholder cards, per-post image uploads
  • Categorised links page, admin-editable via a JSON file in the object store
  • Password-protected admin panel with blog editor and links editor tabs
  • All dynamic content backed by POSIX shell CGI scripts — no Node, no PHP, no runtime
  • S3-compatible storage with atomic local cache mirror for zero-latency reads
  • Mobile-responsive layout with hamburger nav drawer

Challenges

  • BusyBox httpd v1.37 only executes CGI from URLs with /cgi-bin/ as the first path component — scripts at nested paths like /admin/cgi-bin/ are served as static text. Solved with a root-level symlink.
  • mc ls returns exit 0 when any object exists under a bucket prefix, not just the specific key queried — causing every post to appear published on first save. Replaced the S3 existence check with a local filesystem check against the always-in-sync cache.
  • Keeping a manifest JSON file consistent across concurrent saves using only POSIX shell — no locking primitives, no jq.
