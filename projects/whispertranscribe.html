WhisperTranscribe

GPU-accelerated push-to-talk transcription and voice command daemon — one button types, the other commands a local or cloud LLM

View on GitHub

Overview

A Python daemon driven by the two-button MicrophoneController USB HID device. Both buttons share the same hold-to-record flow — audio is captured via PipeWire while the key is held and transcribed by whisper.cpp on release — but diverge in what happens with the result.

Scroll Lock injects the transcription directly into the focused window via ydotool with no clipboard interaction. Pause routes the text through a configurable LLM (local Ollama or Claude CLI) which interprets the spoken instruction and returns an executable action — launching an app, running a shell command in a terminal, or injecting a key sequence.

The daemon grabs the keyboard exclusively via evdev so neither key ever reaches the desktop. A 0.5-second tap guard discards accidental presses. Transcription runs on the GPU via whisper.cpp built with the Vulkan backend, keeping latency low even with the large-v3 model. A systemd user service handles autostart and restart on failure.

Learnings

LLM Command Mode

The LLM receives the transcription and must respond with exactly one of three formats, which the daemon parses and executes:

  • GUI: <cmd> — launches an application or URL silently in the background
  • TERMINAL: <cmd> — runs a shell command in a new terminal window using the configured terminal emulator
  • KEYS: <sequence> — injects raw ydotool keycode events, enabling shortcuts like select-all, undo, or arbitrary key combinations by voice

The Scroll Lock button also doubles as a command trigger: if the transcribed text begins with the configured prefix word (default: command), the remainder is routed through the LLM pipeline instead of being typed.

System Tray

A GTK3 system tray application implements the KDE StatusNotifierItem + DBusMenu protocols, giving a persistent tray icon that reflects the daemon's running state. The tray polls the systemd user service every two seconds and updates the icon between a microphone and a muted microphone accordingly.

The tray menu provides start/stop toggle and a settings dialog covering: keyboard device selection (dropdown of all evdev-capable devices), hotkey capture (click the button then press any key), LLM backend selection (Ollama or Claude), live Ollama model picker (queries the local instance for installed models), and an advanced panel for binary paths, audio backend, language, timeouts, and terminal emulator. Works natively on KDE; GNOME requires the AppIndicator shell extension.

Components

  • key_watcher — evdev device discovery and exclusive keyboard grab; maps multiple keycodes to independent callback pairs
  • recorder — spawns pw-record as a subprocess to stream audio to a temp WAV while the key is held
  • transcriber — invokes whisper.cpp with --no-timestamps -np; strips and returns clean text, or None on failure
  • commander — builds a structured prompt, queries Ollama or Claude CLI, parses the response, and dispatches GUI/TERMINAL/KEYS actions
  • typer — calls ydotool to inject transcribed text or raw key sequences into the active window
  • tray — GTK3 StatusNotifierItem tray with DBusMenu settings dialog
  • daemon — orchestrates all modules; handles SIGTERM/SIGINT cleanly and manages the minimum-duration guard

Work in Progress

The ubuntu-debian-support branch broadens platform compatibility:

  • PulseAudio fallback — recorder.py now auto-detects the available audio backend at startup; uses pw-record on PipeWire systems (Fedora) and falls back to parecord on PulseAudio systems (Ubuntu/Debian). Backend can also be pinned explicitly via config.
  • NVIDIA and CPU whisper.cpp builds — alongside the existing Vulkan/AMD build, the branch documents and supports building whisper.cpp with CUDA for NVIDIA GPUs (-DGGML_CUDA=ON) and a plain CPU-only build for machines without a supported GPU.
  • WAV finalisation fix — recording is now stopped with SIGINT instead of SIGTERM. SIGTERM caused both pw-record and parecord to leave data_size=0 in the WAV header, which whisper.cpp rejects silently. SIGINT triggers a proper flush and correct header write.
  • Keyboard enumeration refactor — device listing extracted into a shared list_keyboard_devices() function, removing duplication between the daemon and the system tray. The tray also gains a clearer error message when StatusNotifierWatcher is unavailable on GNOME.

Hardware

  • MicrophoneController USB HID device — Scroll Lock triggers transcription mode; Pause triggers LLM command mode
  • AMD Radeon RX 7900 XTX — Vulkan backend drives GPU-accelerated whisper.cpp inference (GGML_VULKAN=ON), keeping transcription latency low for both modes
